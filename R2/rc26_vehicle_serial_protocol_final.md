# Rc2026 串口协议定稿

---

## 1. 通信方式

| 项目 | 值 |
|---|---|
| 物理连接 | 双串口全双工：上位机 TX 串口（只发）→ 电控 RX 串口；电控 TX 串口 → 上位机 RX 串口（只收） |
| 控制帧频率 | `50 Hz`（上位机 → 电控），压力大可降到 `30 Hz` |
| 状态帧频率 | `50 Hz`（电控 → 上位机） |
| 波特率 | `115200` |
| 数据位 | `8` |
| 校验 | `None` |
| 停止位 | `1` |
| 多字节整数 | 大端序，高字节在前 |
| 控制帧帧头/帧尾 | `0xCC` / `0xEE` |
| 状态帧帧头/帧尾 | `0xCC` / `0xEE` |

---

## 2. 控制帧：上位机 → 电控

固定长度：`30 bytes`

```text
[0]     0xCC
[1-2]   vx                # int16，前后速度，10000 = 1 m/s，前进为正
[3-4]   vy                # int16，左右速度，10000 = 1 m/s，左移为正
[5-6]   wz                # int16，yaw 角速度，10000 = 1 rad/s，从上往下看逆时针/左转为正
[7-8]   lift_front_target # uint16，前升降目标高度，单位 mm
[9-10]  lift_back_target  # uint16，后升降目标高度，单位 mm
[11]    lift_mode         # uint8，0=normal，1=fast
[12-13] left_arm_pitch1   # int16，左机械臂第 1 pitch，单位 mrad
[14-15] left_arm_pitch2   # int16，左机械臂第 2 pitch，单位 mrad
[16-17] left_arm_pitch3   # int16，左机械臂第 3 pitch，单位 mrad
[18-19] right_arm_pitch1  # int16，右机械臂第 1 pitch，单位 mrad
[20-21] right_arm_pitch2  # int16，右机械臂第 2 pitch，单位 mrad
[22-23] right_arm_pitch3  # int16，右机械臂第 3 pitch，单位 mrad
[24]    left_arm_sucker   # uint8，左机械臂末端吸盘，0=保持，1=打开吸盘/吸气，2=关闭吸盘/松开
[25]    right_arm_sucker  # uint8，右机械臂末端吸盘，0=保持，1=打开吸盘/吸气，2=关闭吸盘/松开
[26]    weapon_pitch      # uint8，武器头 pitch，0=保持，1=平行地面，2=垂直地面
[27]    weapon_gripper    # uint8，武器头夹爪，0=保持，1=打开，2=闭合
[28]    reserved          # uint8，预留，默认 0
[29]    0xEE
```

### 2.1 底盘速度换算

底盘运动方向约定：

```text
+x：车头前方
+y：车体左侧
wz：绕车体竖直轴旋转，从上往下看逆时针为正，也就是车头向左转为正
```

底盘速度：

```text
vx_protocol = vx_mps * 10000
vy_protocol = vy_mps * 10000
wz_protocol = wz_radps * 10000
```

示例：

| 实际速度 | 协议值 |
|---:|---:|
| `+1.0 m/s` | `+10000` |
| `+0.5 m/s` | `+5000` |
| `+0.05 m/s` | `+500` |
| `-0.3 m/s` | `-3000` |
| `+1.0 rad/s` | `+10000` |

速度标定重点：

- 上位机只负责发送 `vx / vy / wz` 协议值，不做四轮解算。
- 电控负责根据全向轮运动学，把 `vx / vy / wz` 解算成四个轮子的目标转速。
- 车速不需要超过 `2 m/s`，因此统一规定 `10000 = 1 m/s`，`10000 = 1 rad/s`。
- 建议电控对 `vx / vy` 做限幅：`±20000` 对应 `±2 m/s`。
- `wz` 可不做额外限幅；若需要限幅，建议最大 `±32000`，对应约 `±3.2 rad/s`。
- 下位机第一版只需要做粗标定，不要求一次性标得非常准。
- 后续上位机会根据里程计实测结果，重新修正 `cmd_vel -> 实际速度` 的比例。

直线速度到轮子转速的初始估算公式：

```text
wheel_rps = wheel_linear_speed_mps / (pi * wheel_diameter_m)
wheel_rpm = wheel_linear_speed_mps / (pi * wheel_diameter_m) * 60
```

其中：

- `wheel_diameter_m` 是全向轮实际直径，单位 m。
- 如果电机还有减速比，需要再乘以减速比换成电机轴转速。
- `wz` 旋转速度需要结合轮子到车体中心的几何尺寸进行解算。
- `vx / vy / wz` 的正方向需要实车验证，若方向相反，优先在电控解算符号里修正。

### 2.2 升降高度

前后两组升降目标高度均为：

```text
uint16，单位 mm
```

说明：

- 最高高度至少 `400 mm`，必须用 `uint16`，不能用 `uint8`。
- 当前协议按前后两组控制，不再单独区分四个轮子。

```text
lift_front_target -> 前两个轮子
lift_back_target  -> 后两个轮子
```

### 2.3 升降模式

```text
lift_mode:
0 = normal
1 = fast
```

说明：

- `normal` 用于武器头拾取、KFS 高度调整这类需要较稳、较准的动作。
- `fast` 用于梅林上下台阶过程中，快速伸出或快速收回，优先节省时间。
- 电控内部建议至少区分速度、加速度和到位容差，不要只改一个速度值。

建议实现：

- `normal`：较小速度、较小加速度、较严格到位容差，用于精确调高。
- `fast`：较大速度、较大加速度、较宽到位容差，用于快速收回和快速伸出。
- 两种模式都仍然是位置闭环，目标都是 `lift_front_target / lift_back_target`，区别只在内部运动参数，不改变协议字段。
- 到达目标附近后，电控都应自动切换到保持状态，避免升降机构来回抖动。
- 建议电控内部把 `fast` 仅作为轨迹参数切换，而不是单独做一套控制逻辑，这样联调更稳。

### 2.4 机械臂角度

两个机械臂共 6 个 pitch，均为：

```text
int16，单位 mrad
1000 = 1 rad ≈ 57.3°
1 mrad ≈ 0.0573°
```

示例：

| 实际角度 | 协议值 |
|---:|---:|
| `0°` | `0` |
| `45°` | `785` |
| `90°` | `1570` |
| `-90°` | `-1570` |
| `-180°` | `-3141` |

#### 协议精度评估

| 项目 | 值 |
|---|---|
| 量化分辨率 | `1 mrad = 0.001 rad ≈ 0.057°` |
| 末端最大位置误差（3 关节误差叠加，臂全展 ~964mm） | **≈ 1.6 mm** |
| 单关节末端误差（425mm 连杆） | ≈ 0.43 mm |

> 结论：**精度足够。** KFS 方块边长数十 mm，吸盘本身也有柔性容差。1.6mm 的最差情况末端误差完全可以接受。实际使用中误差通常远小于此值。

#### 2.4.1 关节编号

每个机械臂 3 个 pitch 关节，从基座到末端：

```
基座 → [pitch1] → link1 → [pitch2] → link2 → [pitch3] → 吸盘
```

| 协议字段 | 关节 | 物理位置 |
|---|---|---|
| `left_arm_pitch1` / `right_arm_pitch1` | pitch1 | 基座旋转关节 |
| `left_arm_pitch2` / `right_arm_pitch2` | pitch2 | 肘关节 |
| `left_arm_pitch3` / `right_arm_pitch3` | pitch3 | 腕关节（吸盘前一个关节） |

#### 2.4.2 角度零点定义（重要）

**当 `pitch1 = pitch2 = pitch3 = 0 mrad` 时，机械臂呈「完全水平前伸」姿态：**

- link1 **水平**，指向车头前方（平行地面）
- link2 与 link1 **共线**（完全打直，无弯折）
- 吸盘轴线与 link2 **共线**（吸盘水平朝前，平行地面）

```
侧视图（从车体左侧往右看）：

  基座 ──── link1 ──── link2 ──── │= 吸盘朝前
  ←────────── 全部水平前伸 ──────────→
```

#### 2.4.3 正方向定义

```
正角度效果：
  pitch1 > 0  →  整臂上抬（远离地面）
  pitch2 > 0  →  肘部上折
  pitch3 > 0  →  吸盘上翘

负角度效果：
  pitch1 < 0  →  整臂下压（靠近地面）
  pitch2 < 0  →  肘部下折
  pitch3 < 0  →  吸盘下压
```

#### 2.4.4 关键姿态对应的角度（参考）

| 姿态 | pitch1 | pitch2 | pitch3 | 吸盘朝向 |
|---|---|---|---|---|
| 全零位（完全前伸，吸盘朝前） | `0` | `0` | `0` | 平行地面，朝前 |
| 吸盘垂直朝下（拾取 KFS 上表面） | 由 IK 解算 | 由 IK 解算 | 由 IK 解算 | 垂直地面，朝下 |

> **给电控**：电控只负责接收这 6 个 `int16 mrad` 值并做角度闭环，**不需要做 IK**。IK 解算全部在上位机完成。电控需要确保：
> 1. 按上述零点定义标定电机编码器零位；
> 2. 按上述正方向定义确定电机旋转正方向；
> 3. 做硬限幅保护（超出 `±180°` 即 `±3141 mrad` 时拒绝或限幅）。

备注：

- 电控只需要按角度闭环，不需要做 IK。
- 上位机会自行计算目标角度，并在上层配置死区。
- 下位机也需要做电机角度硬限幅。例如某个 pitch 只能在 `-180~180 degree` 内运动时，超出范围的目标应被限幅或拒绝执行，避免机械结构撞限位。

### 2.5 机械臂末端吸盘

两个机械臂末端吸盘均为：

```text
uint8
0 = 保持
1 = 打开吸盘 / 吸气
2 = 关闭吸盘 / 松开
```

说明：

- `left_arm_sucker` 对应左机械臂末端吸盘。
- `right_arm_sucker` 对应右机械臂末端吸盘。
- 启动默认值建议为 `0=保持`，避免上位机刚启动时误触发吸气或松开。

### 2.6 武器头 pitch 与夹爪

武器头 pitch：

| 值 | 语义 |
|---:|---|
| `0` | 保持 |
| `1` | 平行地面，拾取武器头状态 |
| `2` | 垂直地面，对接或默认收回状态 |

武器头夹爪：

| 值 | 语义 |
|---:|---|
| `0` | 保持 |
| `1` | 打开 |
| `2` | 闭合 |

## 3. 状态帧：电控 → 上位机

固定长度：`27 bytes`

```text
[0]     0xCC
[1-2]   lift_front_actual    # uint16，前升降当前高度，单位 mm
[3-4]   lift_back_actual     # uint16，后升降当前高度，单位 mm
[5-6]   tof_front_front_mm   # uint16，前升降前侧 ToF 距离，单位 mm
[7-8]   tof_front_back_mm    # uint16，前升降后侧 ToF 距离，单位 mm
[9-10]  tof_back_front_mm    # uint16，后升降前侧 ToF 距离，单位 mm
[11-12] tof_back_back_mm     # uint16，后升降后侧 ToF 距离，单位 mm
[13-14] left_arm_pitch1_now  # int16，左机械臂第 1 pitch 当前角，单位 mrad
[15-16] left_arm_pitch2_now  # int16，左机械臂第 2 pitch 当前角，单位 mrad
[17-18] left_arm_pitch3_now  # int16，左机械臂第 3 pitch 当前角，单位 mrad
[19-20] right_arm_pitch1_now # int16，右机械臂第 1 pitch 当前角，单位 mrad
[21-22] right_arm_pitch2_now # int16，右机械臂第 2 pitch 当前角，单位 mrad
[23-24] right_arm_pitch3_now # int16，右机械臂第 3 pitch 当前角，单位 mrad
[25]    reserved             # uint8，预留，默认 0
[26]    0xEE
```

### 3.1 回传高度

前后两组升降实际高度均为：

```text
uint16，单位 mm
```

说明：

- 电控根据编码器回传当前高度即可。
- 上位机根据 `target - actual` 自己判断是否到位。
- 如果某个轴没有编码器或当前高度无效，可先回传 `0xFFFF` 表示无效。

### 3.2 ToF 回传

ToF 距离：

```text
uint16，单位 mm
```

说明：

- 本版协议支持 4 个 ToF 传感器阻挡高度数据，无需回传 valid 位。
- 如果 ToF 无效或传感器离线，电控可以回传一个固定异常值 `0xFFFF`，上位机按距离阈值处理。
- `tof_front_front_mm` / `tof_front_back_mm` 分别检测前升降两侧的台阶边缘；`tof_back_front_mm` / `tof_back_back_mm` 检测后升降两侧。

### 3.3 机械臂角度回传

两个机械臂当前角度均为：

```text
int16，单位 mrad
```

说明：

- 上位机会根据目标角度、当前角度和死区判断是否到位。
- 如果电控暂时无法回传实际角度，可以先回传当前目标角度，但需要在联调时说明。

### 3.4 预留字段

```text
[25] reserved
```

说明：

- 当前状态帧不回传武器头 pitch 当前状态，也不回传夹爪状态。
- 该字节先预留，默认填 `0`。

## 4. 当前不使用的内容

本版协议暂时不放以下内容：

- 命令序号
- 软件急停
- 底盘使能
- 复杂故障 bit
- checksum
- ACK 应答
- ToF valid 位

原因：

- 急停、使能、遥控器优先级由电控和遥控器侧处理。
- 当前目标是先把上位机和电控联调跑通，协议尽量简单。
- 如果后续串口现场出现误帧、错帧、状态不可判定，再升级 checksum 或状态 bit。

## 5. 电控需要额外注意的事项

1. `vx / vy / wz` 先按 `10000 = 1 m/s`、`10000 = 1 rad/s` 接收。
2. 建议 `vx / vy` 限幅到 `±20000`，对应 `±2 m/s`；`wz` 可不限，或限幅到 `±32000`，对应约 `±3.2 rad/s`。
3. 电控需要根据轮子半径、减速比和全向轮结构，先做一版粗标定，把 `vx / vy / wz` 转成电机转速。
4. 上位机会后续根据里程计重新标定速度比例，所以下位机初版只需要大致准确。
5. 前后两组升降高度目标和回传都用 `uint16 mm`，回传值尽量来自编码器当前高度。
6. `lift_mode` 建议至少支持 `normal` 和 `fast` 两种。
7. `normal` 用于精确调高，`fast` 用于梅林上下台阶中途的快速收回/快速伸出。
8. 两个机械臂 6 个 pitch 都用 `int16 mrad`，电控按角度闭环即可。
9. 每个 pitch 关节下位机都要做角度硬限幅，防止上位机发错角度导致机械撞限位。
10. 武器头 pitch 用 `uint8` 三态枚举：`0=保持`，`1=平行地面`，`2=垂直地面`。
11. ToF 距离用 `uint16 mm`，不单独回传 valid 位。
12. 控制帧固定 `30 bytes`，状态帧固定 `27 bytes`。
13. `fast / normal` 最好实现成同一位置闭环下的两套参数：
    - `normal`：低速度、低加速度、小容差。
    - `fast`：高速度、高加速度、适当放宽容差。
14. 推荐在接近目标时自动减速并进入保持，避免 `fast` 模式下冲过头或反复振荡。
